# 字幕遮挡条 macOS 应用设计规格

## 目标

制作一个可双击运行的 macOS 原生小应用，用一条纯黑、不透明的悬浮横条遮住视频中的中文字幕。横条需要始终显示在其他 App 之上，即使用户把其他 App 放大或切换到 macOS 全屏空间；同时保持足够轻量，启动后无需复杂设置。

## 已确认的用户体验

- 应用形态：独立窗口，不依赖菜单栏操作。
- 外观：纯黑、不透明、无边框横条。
- 初始状态：首次启动时使用接近用户截图的字幕区域尺寸，放在屏幕底部字幕区域。
- 交互：在横条任意位置拖动；从窗口边缘自由调整宽度和高度。
- 关闭：鼠标移入右上角显示关闭按钮；按 `Esc` 也可关闭。
- 持久化：保存上次的位置和尺寸，下次启动恢复。
- 交付：源码和可双击打开的 `.app`。

## 方案选择

### 方案 A：原生 AppKit（采用）

使用 Swift 和 AppKit 创建单窗口应用。AppKit 直接提供窗口层级、全屏空间行为、拖动、键盘事件和窗口缩放能力，最终产物轻量且不需要额外运行时。

### 方案 B：Electron/Tauri

可使用 Web 技术构建控制界面，但对一个单色遮挡条来说运行时和打包体积偏重；窗口置顶和全屏行为还要依赖额外配置。

### 方案 C：浏览器网页窗口

开发成本最低，但无法可靠地覆盖其他 App 或全屏窗口，因此不满足核心需求。

## 架构

- `AppDelegate`：创建应用、恢复或计算初始窗口位置、设置应用退出行为。
- `OverlayWindow`：无边框、可拖动、可缩放的顶层窗口；设置跨 Space 和全屏辅助行为。
- `OverlayView`：绘制纯黑背景，管理鼠标悬停状态，并提供右上角关闭按钮。
- `UserDefaults`：持久化窗口 frame；无可用保存值时按当前屏幕尺寸计算默认 frame。

窗口使用高于普通应用的系统层级，并设置 `canJoinAllSpaces` 与 `fullScreenAuxiliary`，让它在不同桌面空间和全屏应用中保持可见。应用不读取屏幕内容、不连接网络，也不需要辅助功能权限。

## 默认尺寸和位置

首次启动按当前主屏幕动态计算：宽度约为屏幕宽度的 50%，高度约为屏幕高度的 9%，同时限制在合理的最小和最大尺寸范围内；横向居中，垂直位置位于屏幕底部约 10% 的字幕区域。这样在不同分辨率上仍然接近截图中的比例。

## 交互细节

- 黑色区域不显示标题栏或常驻工具栏，避免遮挡字幕之外的画面。
- 横条整体可拖动；窗口边缘提供系统缩放区域。
- 右上角关闭按钮默认隐藏，鼠标进入右上角热区后出现；鼠标离开后淡出。
- `Esc` 关闭窗口并结束应用。
- 关闭按钮和键盘关闭路径都走同一个应用退出流程，并在退出前保存 frame。
- 保存的 frame 如果超出当前屏幕或当前显示器配置发生变化，则自动夹回可见区域。

## 错误处理和边界情况

- 没有保存过窗口位置时使用默认 frame。
- 保存的 frame 解析失败或尺寸低于最小值时丢弃并重新计算。
- 多显示器切换后，窗口保留用户拖动后的位置；若完全不可见，则移动到主屏幕可见区域。
- 应用只提供黑色遮挡，不处理视频、字幕识别或自动跟踪；用户负责手动调整位置和大小。

## 验证范围

1. 使用 Swift 编译并生成可双击运行的 `.app`。
2. 启动后确认默认尺寸和位置接近规格。
3. 将其他 App 放大或切换全屏，确认遮挡条仍在最上层。
4. 验证拖动、四边缩放、右上角关闭按钮和 `Esc` 关闭。
5. 退出后重新启动，确认位置和尺寸被恢复。
6. 验证窗口移动到屏幕边缘或多显示器场景时仍保持可见。

## 非目标

本版本不包含颜色/透明度设置、自动识别字幕位置、全局快捷键、多个遮挡条、菜单栏图标或登录启动项。这些可以在后续版本加入，但不影响当前的核心使用流程。
